操作指南：
1. Programmer assistor can help you to understand source code more easily and quickly by generating some topics with the help of Mallet.
2. What you need to do:
A. Put all source code and javadoc into a folder and upload it. Programmer assistor will do preprocessing. Then output a MALLET import data directory and a folder that has preprocessed files. This folder is the input for topic modeling by MALLET
B. Topic modeling for MALLET.
	(a) Open bash and enter MALLET directory.
	(b) Importing data: 
		$ ./bin/mallet import-dir --input MALLET-import-data-directory --output ./preprocess.mallet --keep-sequence --remove-stopwords
	(c) Training topics: 
		$ ./bin/mallet train-topics --input ./preprocess.mallet --num-topics 45 --num-top-words 10 --num-iterations 10000 --optimize-burn-in 1000 --optimize-interval 100 --output-state ./topic-state.gz --output-topic-keys ./keys.txt --output-doc-topics ./composition.txt --xml-topic-phrase-report ./topic-phrases.xml --word-topic-counts-file ./word_top.txt
	(d) If the work will done, There will be 5 files in the directory of Programmer Assiter. Then you need to input these paths in show page.
C. Generate views for topics: Input reletive path of file that generated by MALLET and click "Start Showing" button. You can get several styles of topics showing.
